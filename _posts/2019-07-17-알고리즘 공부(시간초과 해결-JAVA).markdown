Stack 관련된 문제를 풀다가 어려움에 직면하였다.
문제에 주어진 예제는 모두 해결하는 알고리즘을 짰으나 문제에서 요구하는 제한시간을 통과하지 못하였다.
아직도 해결하지 못해서 내 정신상태가 이상해지는 것 같다.

<a href="https://www.acmicpc.net/problem/1874">[백준 1874번]</a>

이와 관련해서 다른 사람들도 비슷한 문제로 해결을 못하는 것을 파악했고
질문에 달린 답변들을 관찰하다가 입출력의 방식에 따라 시간초과가 날수있다는 사실을 알수 있었다.
이와 관련된 백준 문제는

<a href="https://www.acmicpc.net/problem/15552">[백준 15552번]</a>


### JAVA - Scanner를 통한 입출력

> <pre>
> import java.util.Scanner;
> 
> public class Main {
> 	public static void main(String[] args) {
> 		Scanner sc = new Scanner(System.in);
> 		int T = sc.nextInt();// 테스트케이스의 개수 T(최대 1000,000)
> 		for (int i = 0; i < T; i++) {
> 			int A = sc.nextInt();
> 			int B = sc.nextInt();
> 
> 			System.out.println(A + B);
> 		}
> 	}
> }
> </pre>

### JAVA - BufferedReader, BufferedWriter 를 통한 입출력

> <pre>
> import java.io.*;
> import java.util.StringTokenizer;
> 
> public class Main {
> 	public static void main(String[] args) throws IOException {
> 		BufferedReader bf = new BufferedReader(new InputStreamReader(System.in));
> 		BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
> 
> 		int T = Integer.parseInt(bf.readLine());// 테스트케이스의 개수 T(최대 1000,000)
> 		for (int i = 0; i < T; i++) {
> 			String s = bf.readLine();
> 			StringTokenizer st = new StringTokenizer(s);
> 			int A = Integer.parseInt(st.nextToken());
> 			int B = Integer.parseInt(st.nextToken());
> 			bw.write(String.valueOf(A+B+"\n"));
> 		}
> 		bw.flush();
> 		bw.close();
> 	}
> }
> </pre>
